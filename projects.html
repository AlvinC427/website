Things I've Built

Here's a collection of fun academic and personal projects I've worked on. This includes everything from small experiments to full applications along with my thoughts on them.

  • Ascension: A Foddian-style game developed during my time in Drexel's CI 102 & 103 courses.
    See Code on GitLab | Wiki | Watch Demo

    • üéÆ Ascension Demo Video


    In case you were wondering, Foddian games are a genre of games designed to be frustratingly tough yet addicting with precision and patience being everything. What made this project valuable wasn't just the final product, but everything I learned in the process of getting there.

    This was one of my first times working on a larger, shared codebase where I had to read, understand, and build on other people's work. We used GitLab for version control, and I quickly picked up how to manage branches, resolve merge conflicts, and keep our workflow organized so we didn't accidentally overwrite each other's progress. A lot of it was new to me; from figuring out how to set up Pygame environments to troubleshooting obscure bugs we'd occasionally introduce in each other's code, I learned to communicate changes clearly and build on unfamiliar code with confidence.

    Throughout the project, I made sure to cleanly document my sections so teammates could easily follow what I was doing. It became an easy routine over time and a great exercise in both technical problem-solving and collaborative development.

    Aside from building the gameplay mechanics and physics, we also took time to polish small details like custom sprites and sound effects. We even created a demo video that I hope you check out! The project taught me a lot about structuring game loops, learning a new Python library, and the chaotic fun of balancing difficulty curves in platformers.

    Tech/Tools Used: Python, Pygame, GitLab, Thonny (IDE), Agile Development, Scrum, Kanban board
    January 2025 - June 2025

  • Frog Game: A small side project built to practice object-oriented programming in Python.
    See the Code on GitLab

    • üê∏ Frog Game Demo Video


    This game is an arcade-style game I built on my own to practice object-oriented programming in Python. It started as a late-night idea, and I ended up coding most of it before I went to bed. The project focuses on fundamental OOP concepts like class design, inheritance, and managing object interactions within a Pygame loop.

    Beyond being a fun standalone project, it ended up being a valuable stepping stone towards the end of the term while working on Ascension. The experience I gained building out game loops, handling sprite behaviors, and structuring clean, reusable code directly carried over into that much larger team project.

    Tech Used: Python, Pygame
    May 2025

    Earlier (but still good) Projects

  • Virtual Pest Project: A command-line and browser-based state management program.

    • This was the final project for Drexel's Intro to CS course where we had to learn and use a custom programming language called Tranquility to design operational state transitions for a virtual pet simulation. While the visual interface wasn't straightforward with some buttons and images, the focus of the project was on project planning and applying logical thinking to structured state management using a finite state machine.

    Although many students in the course dismissed the language and project as overly niche, I thought it was a valuable exercise in learning how to work with unfamiliar tools and staying focused on core programming concepts. The project helped enhance my programming mindset and appreciation for language-agnostic problem-solving.

    Tech Used: CLI, HTML/CSS Tranquility
    November 2024 - December 2024

  • John's Farmer Market: My very first coding project - a simulated online shopping experience.
    About the AP CSP Performance Task

    • This was the first program I ever built. With 2 friends, we developed this as part of our AP Computer Science Principles exam in high school. The project simulated an online farmer's market with a basic checkout system, interactive price adjustments, and dynamic item lists. It was built during 9 hours of in-class time for the AP exam. This included coding, recording a demo video, and writing up a technical explanation (unfortunately none I can find).

    This project introduced me to programming fundamentals like control structures, variables, user interaction, and debugging. It's what sparked my interest in software and web development, giving me a sense of what it's like to build something from scratch.

    Tech Used: JavaScript, HTML, CSS, Replit
    April 2022
